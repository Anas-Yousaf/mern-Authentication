1. Introduction to Authentication and Authorization

Authentication

Authentication verifies WHO the user is. It's like showing your ID card.

  • Login with email/password
  • Check if user exists in database
  • Compare entered password with stored password

Authorization

Authorization determines WHAT the user can do. It's like checking permissions.

  • Admin can delete posts
  • Regular user can only view posts
  • Guest cannot access dashboard
Simple Rule: Authentication = Login, Authorization = Permissions

3. Password Hashing with bcrypt

Why Hash Passwords?

Never store plain text passwords! Hashing makes them unreadable even if database is stolen.

bcrypt Basic Usage

// Hash password before saving const bcrypt = require('bcryptjs'); const hashedPassword = await bcrypt.hash(password, 10); // Compare password during login const isMatch = await bcrypt.compare(password, hashedPassword);
Remember:
  • bcrypt.hash() - to encrypt password
  • bcrypt.compare() - to check password
  • Always use await with bcrypt methods

4. Complete Single File Implementation

Complete Backend Code (One File)

app.js
const express = require('express'); const mongoose = require('mongoose'); const bcrypt = require('bcryptjs'); const app = express(); // Connect to MongoDB mongoose.connect('mongodb://localhost:27017/authdb'); // User Schema const User = mongoose.model('User', { name: String, email: String, password: String }); // Middleware app.use(express.urlencoded({ extended: true })); app.set('view engine', 'ejs'); // Routes app.get('/', (req, res) => { res.render('login'); }); // Register Route app.post('/register', async (req, res) => { const { name, email, password } = req.body; // Hash password const hashedPassword = await bcrypt.hash(password, 10); // Save user const user = new User({ name: name, email: email, password: hashedPassword }); await user.save(); res.send('User registered successfully!'); }); // Login Route app.post('/login', async (req, res) => { const { email, password } = req.body; // Find user const user = await User.findOne({ email: email }); if (!user) { return res.send('User not found'); } // Check password const isMatch = await bcrypt.compare(password, user.password); if (isMatch) { res.send(`Welcome ${user.name}!`); } else { res.send('Wrong password'); } }); app.listen(3000, () => { console.log('Server running on port 3000'); });

Even Shorter Version (Minimal Code)

minimal-auth.js
const express = require('express'); const mongoose = require('mongoose'); const bcrypt = require('bcryptjs'); const app = express(); mongoose.connect('mongodb://localhost:27017/authdb'); const User = mongoose.model('User', { name: String, email: String, password: String }); app.use(express.urlencoded({ extended: true })); app.use(express.static('public')); // Register app.post('/register', async (req, res) => { const hashedPassword = await bcrypt.hash(req.body.password, 10); const user = new User({ name: req.body.name, email: req.body.email, password: hashedPassword }); await user.save(); res.send('Registered!'); }); // Login app.post('/login', async (req, res) => { const user = await User.findOne({ email: req.body.email }); if (user && await bcrypt.compare(req.body.password, user.password)) { res.send(`Welcome ${user.name}!`); } else { res.send('Login failed!'); } }); app.listen(3000);
For Exams:
  • Use the minimal version (30 lines of code)
  • Focus on: connect DB → create schema → hash password → compare password
  • Don't worry about error handling or validation
  • Keep HTML forms simple with basic input fields
